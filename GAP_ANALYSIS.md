## ~~ОШИБКА 1: Двойной расчёт кислорода (архитектурная)~~ ✅ ИСПРАВЛЕНО (2026-03-11)

**Файл:** `human_development_module/src/inducers.rs:59-66` + `lib.rs:337-343`

**Проблема:** В коде сосуществуют ДВЕ формулы кислорода:

**Старая (неправильная)** — `centrosomal_oxygen_level()` в `inducers.rs`:
```rust
mito_shield = 1.0 - ros×0.50 - aggregates×0.30 - carbonylation×0.20
o2 = 1.0 - mito_shield  // = ros×0.5 + aggregates×0.3 + carbonylation×0.2
```

**Новая (правильная)** — `mitochondrial_module/src/lib.rs:206-211`:
```rust
mito_shield_contribution = fusion_index×0.40 + membrane_potential×0.35 + (1-ros_production)×0.25
```

**Как они комбинируются в `apply_oxygen_detachment()`:**
```rust
let raw_oxygen = centrosomal_oxygen_level(&comp.centriolar_damage); // СТАРАЯ формула
let oxygen = (raw_oxygen * (2.0 - mito_shield)).min(1.0);           // ПЕРЕМНОЖЕНИЕ
```

**Почему это неправильно:**
1. Карбонилирование (`protein_carbonylation`) центриоли попадает в расчёт кислородного щита — это причинно-следственная инверсия. Карбонилирование — СЛЕДСТВИЕ O₂, не причина ослабления щита.
2. Когда оба модуля работают, ros_level влияет ДВАЖДЫ:
   - через `centrosomal_oxygen_level` (ros×0.5)
   - косвенно через митохондриальный модуль → ros_production → mito_shield_contribution
3. В молодой здоровой клетке с низкими повреждениями: `raw_oxygen ≈ 0.006` → кислород практически не достигает центриолей → O₂-путь отщепления почти не работает. Вся нагрузка на PTM-путь.

**Правильное решение:**
```rust
// Когда MitochondrialModule активен — использовать ТОЛЬКО его формулу:
fn apply_oxygen_detachment(comp, mito_shield, rng) {
    let o2_at_centriole = (1.0 - mito_shield).clamp(0.0, 1.0);
    // НЕ raw_oxygen × (2.0 - mito_shield)
    // mito_shield_contribution уже учитывает ros_production из мтДНК
    if o2_at_centriole > 0.01 {
        detach_by_oxygen(&mut comp.inducers, o2_at_centriole, age, rng);
    }
}
```

## ~~ОШИБКА 2: base_detach_probability — расхождение со спецификацией~~ ✅ ИСПРАВЛЕНО (2026-03-11)

**Файл:** `cell_dt_core/src/components.rs:385`

**Спецификация** (`cdata_inducer_correction.md`):
```
base_detach_probability: 0.1   // <- Canon
mother_bias: 0.6
age_bias_coefficient: 0.003
```

**Текущий код:**
```rust
base_detach_probability: 0.002,   // <- 50× меньше спека!
mother_bias: 0.5,                 // <- меньше спека
age_bias_coefficient: 0.0,        // <- нет возрастного сдвига
```

**Следствие:**
С `base_detach_probability=0.002` и типичным O₂≈0.006 в молодой клетке:
- `mother_prob = 0.006 × 0.002 × 0.5 = 0.000006/шаг`
- За 70 лет (25 550 шагов): **~0.15 детачментов** через O₂-путь
- Модель фактически игнорирует O₂-путь, хотя по теории он ПЕРВИЧЕН

С правильным `0.1` и правильной формулой O₂ (`1 - mito_shield`):
- В молодой клетке `mito_shield ≈ 0.95` → O₂≈0.05
- `mother_prob = 0.05 × 0.1 × 0.6 = 0.003/шаг`
- За 70 лет: **~77 детачментов** — биологически реалистично для M=10

**Результат:** После исправления формулы O₂ и рекалибровки — `base_detach_probability = 0.0003`
дает lifespan = 78.4 лет (цель: ~78 лет ✅). Смерть от damage threshold (0.735 ≈ 0.75).

## ~~ОШИБКА 4: Нет ткань-специфичной асимметрии M→stem vs D→stem~~ ✅ ИСПРАВЛЕНО (2026-03-11)

**Источник:** Papers `20260127_Centrioles as determinants`, `20260127_Centrioles as intracellular timers`

Бумаги описывают два противоположных паттерна:
- **Drosophila germline, радиальная глия млекопитающих:** материнская → стволовая клетка, дочерняя → дифференцирующая
- **Drosophila нейробластов:** НАОБОРОТ — дочерняя → стволовая (исключение!)
- **Mammalian hematopoietic:** материнская центриоль → сторона апикального полюса (LT-HSC)

Текущая модель: одна универсальная логика для всех тканей.

**Действие:** Добавить в `TissueType` поле `asymmetry_rule: CentrioleAsymmetryRule`:
```rust
pub enum CentrioleAsymmetryRule {
    MotherToStem,   // germline, radial glia, HSC
    DaughterToStem, // Drosophila neuroblast
    Symmetric,      // epithelium, default
}
```
И учитывать в `asymmetric_division_module`.

## ОШИБКА 5: Centrosomal Memory не реализована

**Источник:** Papers `20260205_Centrosomal Memory`, `20260115_Strategic Timekeepers`

**Концепция:** Материнская центриоль несёт **центросомальную память** — высокоразмерный вектор состояния V(t), кодирующий историю клетки:
- PTMome: паттерны модификаций (не 4 скаляра, а паттерны)
- История стресса (окислительного, температурного)
- Метаболический статус (ATP/ADP ratio, Ca²⁺)
- Ориентация в нише (позиция относительно базальной мембраны)

Текущая модель использует только 4 скаляра PTM. Это упрощение нормально для v1, но важно обозначить.

**Действие:** Добавить в компонент `CentriolePair.mother` поле `centrosomal_memory: CentrosomeMemo` (отдельная структура с history_vector [f32; 8] или аналог) — как расширение для v3.

## ОШИБКА 6: CAFD-механизмы не моделируются (частичное пропускание)

**Источник:** Paper `20260214_Centriole-Associated Fate Determinants (CAFDs)`

Статья разделяет индукторы на **5 функциональных классов**:
1. **Segregational** — активно сегрегируются при делении (Prospero, Numb, aPKC)
2. **Sensory** — рецепторы нишевых сигналов (IFT-proteins, PTCH1 на реснице)
3. **Stored/Latent** — депонированы до активации сигналом (STAT3, Gli2)
4. **Locally Synthesized** — mRNA транслируется прямо у центросомы (Plp, nanos)
5. **Transport-Mediated** — накапливаются через моторные белки (dynein cargo)

Текущая модель: один пул индукторов (M-set и D-set), без разделения на классы.

**Практическое следствие:** Нельзя смоделировать ситуацию, когда Sensory-CAFD потерян (ресничка), но Segregational-CAFD цел → клетка всё ещё делится асимметрично, но не получает нишевые сигналы. В текущей модели потеря центриолярных молекул = единый счётчик.

**Действие:** Разделить `CentrioleInducerSet` на несколько sub-pools — или задокументировать это как v3 feature.

## ПРОПУЩЕННЫЕ ФИЧИ (не ошибки, а отсутствующие модули)

### A. Элиминация центриолей (путь к тотипотентности)
**Источник:** Paper `202502_Centriole Elimination: A Mechanism for Resetting Entropy`

В яйцеклетках центриоли материнского происхождения **элиминируются** при оплодотворении → сброс центросомальной памяти → тотипотентность зиготы. Это механизм "репрограммирования".

Применение:
- Объясняет, почему зигота тотипотентна — не просто из-за M+D=full, но из-за de novo сборки
- Предсказывает: фармакологическая элиминация центриолей → временная деполяризация → повышение потентности (терапевтический таргет)
- Текущий код: элиминация не предусмотрена, только потеря индукторов

### B. De novo сборка центриолей
**Источник:** Paper `20260131_De novo centriole formation`

В клетках без центриолей (ооциты, некоторые раковые клетки) возможна де ново сборка. Это:
- Объясняет, почему некоторые раковые клетки "вечно молоды"
- Предсказывает: de novo centriole с fresh M+D → временная тотипотентность
- Текущий код: нет de novo pathway

### ~~C. Фармакологический контроль судьбы клетки~~ ✅ РЕАЛИЗОВАНО (2026-03-11)
**Источник:** Paper `20260216_Pharmacological Control via CAFDs`

Статья описывает 4 класса таргетируемых вмешательств:
1. **CAFD-retainers** — молекулы, стабилизирующие индукторы (конкурируют с O₂)
2. **CAFD-releasers** — принудительное отщепление → ускоренная дифференцировка (онкология)
3. **Centrosome cargo inducers** — загрузка новых CAFDs из экзогенного источника
4. **O₂-shield boosters** — митохондриальные препараты (NAD+, urolithin A)

Текущие интервенции: Senolytics, NadPlus, CR, Tert, Antioxidant — только частично покрывают это.

**Отсутствует:**
- CAFD-retainer (прямая стабилизация индукторов)
- CAFD-releaser (контролируемое ускорение дифференцировки)
- Centrosome cargo loader (внешние индукторы)

### D. Трансплантация центросом
**Источник:** Paper `20260116_Centrosome Transplantation`

Ключевой эксперимент, предложенный теорией: трансплантация молодой центросомы в старую клетку → омоложение?
Если M и D комплекты переносятся → потентность восстанавливается?

Текущий код: нет компонента для трансплантации. Нужен `TransplantIntervention`.

### E. Органоидный цифровой двойник
**Источник:** Paper `20260218_Digital Twins of Cell Differentiation`

Статья предлагает многоклеточный цифровой двойник с клеточными слоями:
- Layer 0: LT-HSC (long-term hematopoietic stem cells)
- Layer 1: ST-HSC → Layer 2: MPP → Layer 3: committed progenitors → Layer 4: mature cells
- Каждый слой — отдельная популяция с собственными CDATA-параметрами

Текущий код: одна ниша = одна сущность. Иерархии нет.

## РЕЗЮМЕ ДЛЯ АВТОРА

**Модель ПРАВИЛЬНО:**
- Воспроизводит старение по правильному механизму (CDATA)
- Архитектурно расширяема
- Тесты существуют (57 штук), калибровка проведена

**Модель НЕПРАВИЛЬНО:**
1. **Формула O₂** — комбинируются две разные формулы, что создаёт двойной счёт
2. **base_detach_probability** — значение в 50 раз ниже, чем в спецификации → O₂-путь практически выключен, модель живёт на PTM-пути

**Практический эффект:** Модель воспроизводит правильный lifespan, но по ДРУГОЙ траектории, чем должна. O₂-механизм (первичный) почти не работает, PTM-механизм (вторичный) делает всю работу. Это нужно исправить ДО публикации.
