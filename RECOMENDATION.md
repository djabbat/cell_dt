# Анализ и рекомендации: Cell DT

## 1. Общая оценка

**Cell DT** — хорошо спроектированная Rust-платформа для симуляции клеточного старения на основе теории CDATA. Архитектура корректная, ядро CDATA реализовано полностью. Основная проблема: ~3 из 6 модулей являются заглушками, и модули работают независимо друг от друга.

---

## 2. Статус модулей

| Модуль | Статус | Замечание |
|--------|--------|-----------|
| `human_development_module` | ✅ Полный | Ядро CDATA работает, калибровка ~78 лет |
| `cell_cycle_module` | 🟡 Частичный | Прогрессия фаз OK, чекпоинты не форсируются |
| `transcriptome_module` | 🟡 Частичный | Экспрессия генов, TF, пути — нет обратной связи с циклом |
| `centriole_module` | 🔴 Заглушка | `step()` пустой |
| `asymmetric_division_module` | 🔴 Заглушка | Нет событий деления |
| `stem_cell_hierarchy_module` | 🔴 Заглушка | Нет логики пластичности |

---

## 3. Критические пробелы и рекомендации

### 3.1 CentrioleModule — реализовать `step()`

**Проблема:** Модуль имеет параметры (`acetylation_rate`, `oxidation_rate`), но `step()` пустой — повреждения не применяются к ECS-компонентам.

**Рекомендация:** В `step()` итерировать по всем сущностям с `CentriolarDamageState` и применять инкременты:

```rust
for (_, damage) in world.query_mut::<&mut CentriolarDamageState>() {
    damage.protein_carbonylation += self.params.oxidation_rate * dt;
    damage.acetylation += self.params.acetylation_rate * dt;
    // ... и т.д.
}
```

Это позволит использовать `CentrioleModule` независимо от `HumanDevelopmentModule`.

---

### 3.2 CellCycleModule — форсировать чекпоинты

**Проблема:** Чекпоинты (`G1SRestriction`, `SpindleAssembly`, и др.) распознаются, но клетка никогда не арестуется и не уходит в апоптоз.

**Рекомендация:** Добавить проверку условий арреста при переходе фаз:

```rust
// При переходе G1 → S
if dna_damage > threshold {
    // Активировать чекпоинт, не переходить в S
    state.arrested = true;
    return;
}
```

Без этого симуляция клеточного цикла не имеет биологического смысла при стрессе.

---

### 3.3 Межмодульная интеграция

**Проблема:** Модули не общаются между собой. Транскриптом не влияет на клеточный цикл, центриолярные повреждения из `CentrioleModule` не видны `HumanDevelopmentModule`, и т.д.

**Рекомендация — три приоритетных связи:**

| От | К | Что передавать |
|----|---|----------------|
| `TranscriptomeModule` | `CellCycleModule` | Уровень Cyclin D/E → длительность G1 |
| `CentrioleModule` | `HumanDevelopmentModule` | Агрегаты/карбонилирование → `CentriolarDamageState` |
| `CellCycleModule` | `TranscriptomeModule` | Фаза цикла → экспрессия E2F, MYC |

Реализовать через shared ECS-компоненты (уже есть `CentriolarDamageState`, `CellCycleStateExtended`) — данные записываются одним модулем, читаются другим.

---

### 3.4 AsymmetricDivisionModule — логика делений

**Проблема:** Нет фактического создания дочерних сущностей при делении клетки.

**Рекомендация:** Реализовать очередь событий деления:

```rust
struct DivisionEvent {
    parent: Entity,
    division_type: DivisionType, // Symmetric / Asymmetric
    niche_id: u32,
}
```

В `step()`: собирать кандидатов (высокий `stemness_potential`, фаза M), создавать новые сущности с нужными компонентами, убывать S-индукторы.

---

### 3.5 StemCellHierarchyModule — пластичность

**Проблема:** Потентность отслеживается, но никогда не изменяется.

**Рекомендация:** Добавить переходы потентности на основе TF-уровней из `TranscriptomeState`:

```
NANOG + OCT4 + SOX2 высокие → потентность растёт (дедифференцировка)
Все три низкие → потентность снижается (коммитмент)
```

---

## 4. Научная корректность CDATA-параметров

### Плюсы:
- Калибровка `DamageParams::default()` корректно воспроизводит ~78-летний LifeSpan
- ROS feedback-петля с коэффициентом 0.12 отражает реальную амплификацию оксидативного стресса
- Предел Хейфлика (~50 делений) реализован через S-индукторы
- Мультипликатор ×1.6 после 40 лет соответствует концепции антагонистической плейотропии
- Варианты `progeria()` (×5) и `longevity()` (×0.6) реалистичны

### Спорные моменты:
- `sasp_onset_age: 45.0` — в литературе SASP часто описывается как постепенный процесс, а не ступенчатый
- `senescence_threshold: 0.75` — бинарная смерть при пересечении порога не отражает гетерогенность популяции клеток; стоит рассмотреть вероятностную функцию смерти
- Appendage loss rates (CEP164: 0.0113) — нет прямой ссылки на экспериментальные данные; стоит зафиксировать источники в комментариях

---

## 5. Производительность

- Текущая архитектура поддерживает 1000+ сущностей параллельно (Rayon)
- `VisualizationManager` хранит историю до 1000 снапшотов — при долгих симуляциях может быть узким местом; рассмотреть ring buffer
- `TranscriptomeModule` при мутациях создаёт накладные расходы — проверить, что `rand::thread_rng()` не создаётся каждый шаг

## 6. Приоритизированный план доработки

### Этап 1 — Функциональные заглушки (высокий приоритет)
1. Реализовать `CentrioleModule.step()` с применением damage rates к ECS
2. Форсировать чекпоинты в `CellCycleModule` (арест при повреждении ДНК)

### Этап 2 — Межмодульная интеграция (высокий приоритет)
3. Транскриптом → Клеточный цикл (Cyclin D/E → длительность G1)
4. CentrioleModule → HumanDevelopmentModule (синхронизация `CentriolarDamageState`)

### Этап 3 — Деления и иерархия (средний приоритет)
5. Реализовать события деления в `AsymmetricDivisionModule`
6. Добавить TF-зависимую пластичность в `StemCellHierarchyModule`

### Этап 4 — Валидация и документация (низкий приоритет)
7. Добавить ссылки на литературу для каждого параметра `DamageParams`
8. Написать интеграционные тесты для полного lifecycle (Zygote → Death)
9. Python биндинги для анализа в Jupyter

## 7. Итог

Проект имеет сильную научную основу и корректную архитектуру. **Ядро CDATA готово к использованию.** Следующий приоритет — превратить заглушки в рабочие модули и наладить межмодульную коммуникацию через ECS. После этого платформа станет полноценным инструментом для исследований клеточного старения.
